# Starting Grasshopper（启动 grasshopper）

# 启动

Grasshopper is an add-on or plugin for Rhino. It is part of the default Rhino 6+ installation. This is a link to the official Grasshopper website. This site offers a lot of tutorials, forums and links to other websites.

Starting Grashopper can be done by typing grasshopper in the Rhino command line. Grasshopper will open on top of Rhino. You can also click on the Grasshopper icon in the toolbar.

Grasshopper 是 Rhino 的附加组件或插件。它是默认 Rhino 6+ 安装的一部分。这是官方 Grasshopper 网站的链接。该站点提供了许多教程、论坛和其他网站的链接。

可以通过在 Rhino 命令行中输入 Grasshopper 来启动 Grasshopper。Grasshopper 将在 Rhino 顶部打开。您也可以单击工具栏中的 Grasshopper 图标。

# Introduction（介绍）

# 脚本

Grasshopper is an add-on for Rhino which offers explicit history modeling. This term requires some explanation:

Rhino doesn’t always keep track on how the model was constructed. Some other 3D modeling applications ‘remember’ for example from which set of curves a surface was created. When you change these curves, the surface will change accordingly as if it was newly created. This is called a history. Software such as Maya and 3DS Max implement the concept of history. It supports the design process by enabling you to go back to earlier steps of the modeling process and change them, thereby changing the end result. Rhino is limited in its support for history.

Grasshopper 是 Rhino 的一个附加组件，它提供了明确的历史建模。这个术语需要一些解释：

Rhino 并不总是跟踪模型的构建方式。其他一些 3D 建模应用程序 “记住” 例如从哪组曲线创建曲面。当您更改这些曲线时，曲面将相应更改，就好像它是新创建的一样。这叫做历史。Maya 和 3DS Max 等软件实现了历史的概念。它通过使您能够返回到建模过程的早期步骤并更改它们来支持设计过程，从而改变最终结果。Rhino 对历史的支持有限。

# 显式历史

To address this, the concept of Explicit History was developed for Rhino. This formed the basis of what is now the Grasshopper plug-in. Explicit history is clearly visible in Grasshopper. As you may notice when you start building your scripts, the steps you take to get the final result are saved intermediately, and can be re-used for other goals.

为了解决这个问题，为 Rhino 开发了显式历史的概念。这构成了现在的 Grasshopper 插件的基础。明确的历史在 Grasshopper 中清晰可见。当您开始构建脚本时，您可能会注意到，您为获得最终结果所采取的步骤会在中间保存，并且可以重新用于其他目标。

一个盒子被移到右边，但原来的盒子仍然可见：显式历史

# 脚本的结果

Explicit History differs from the traditional modeling history in that you explicitly construct the History and thereby the object instead of being the result of a set of commands generating an object. This can be effective in the design process as it enables you to specify complex relationships between components.

Each Grasshopper definition consists of a network of components connected using connections and with sliders as input. Grasshopper uses the Rhino viewport to display the result of the definition.

It is recommended to now take a look at the Introduction Grasshopper Geometry and Grasshopper Interface.

显式历史与传统建模历史的不同之处在于您显式地构建历史，从而构建对象，而不是生成对象的一组命令的结果。这在设计过程中非常有效，因为它使您能够指定组件之间的复杂关系。

每个 Grasshopper 定义都由一个组件网络组成，这些组件使用连接和滑块作为输入进行连接。Grasshopper 使用 Rhino 视口来显示定义的结果。

建议现在看一下 Grasshopper Geometry 和 Grasshopper Interface 介绍。

# List of important terminology (重要术语列表)

The Grasshopper Toipedia uses some specific terminology. Here is a brief explanation of the most important terms that are used throughout this article:

  • Component: a building-block that performs a specific action
  • Parameter: a special component that contains data (values) - instead of performing an action
  • Number Slider: a utility that allows the user to choose - - numeric values using a slider
  • Panel: a utility that allows the user to define or display numeric values or text
  • Definition: a network of grasshopper components - your Grasshopper ‘model’

Grasshopper Toipedia 使用了一些特定的术语。以下是对整篇文章中使用的最重要术语的简要说明：

  • 组件：执行特定操作的构建块
  • 参数：包含数据（值）而不是执行动作的特殊组件
  • Number Slider：允许用户使用滑块选择数值的实用程序
  • 面板：允许用户定义或显示数值或文本的实用程序
  • 定义：蚱蜢组件网络 - 您的蚱蜢 “模型”
